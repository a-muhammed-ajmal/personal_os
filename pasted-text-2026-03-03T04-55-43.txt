🗄️ PART 1: SUPABASE DATABASE SCHEMA (RELATIONAL)

Create the following relational database schema in Supabase. Ensure all foreign keys are correctly linked to enable seamless data fetching.

1. projects Table

id: UUID (Primary Key)

name: String

area_of_life: Enum ('Personal', 'Professional', 'Financial', 'Health & Wealth', 'Relationships', 'Vision & Mission', 'Other')

status: Enum ('Active', 'On Hold', 'Completed')

created_at: Timestamp

2. tasks Table

id: UUID (Primary Key)

title: String

description: Text

due_date_time: Timestamp

tags: Array of Strings

reminder: Timestamp

priority: Enum ('Low', 'Medium', 'High')

location: String

deadline: Timestamp

file_url: String (Supabase Storage link)

project_id: UUID (Foreign Key -> projects.id)

parent_task_id: UUID (Foreign Key -> tasks.id, for Subtasks)

is_completed: Boolean (Default: false)

created_at: Timestamp

3. habits Table (The Definitions)

id: UUID (Primary Key)

name: String

category: Enum ('Morning', 'Midday', 'Evening')

created_at: Timestamp

4. habit_logs Table (The Daily Tracking)

id: UUID (Primary Key)

habit_id: UUID (Foreign Key -> habits.id)

log_date: Date

is_completed: Boolean (Default: false)

5. notes_resources Table

id: UUID (Primary Key)

title: String

content: Text

type: Enum ('Note', 'Resource')

project_id: UUID (Foreign Key -> projects.id)

file_url: String (Supabase Storage link)
